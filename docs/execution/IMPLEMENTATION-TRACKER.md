# Implementation Tracker (IMPL-TRACKER-v1)

## 1. The Project Owner operating model (how a sprint actually moves)
The **Project Owner (PO)** owns the loop. Worker agents BUILD; they **never decide when work is finished.** Per sprint the PO: **plans** scope → **dispatches** worker agent(s) → **monitors** intermediate output → **requests evidence** → **interrupts/redirects** on deviation from the goal → **validates the result against the Definition of Done** → and only then **allows completion**. Verification is independent (author≠verifier, DRB §6): the builder agent and the verifier run on a **distinct execution substrate** with a **class-appropriate adversarial probe re-run on neutral CI hardware**, and the merge approver is an **identity ≠ the PR author** (founder as reviewer-of-last-resort).

**Sprint state machine (every sprint):**
`NOT STARTED → IN PROGRESS → VERIFY → FOUNDER REVIEW (only where flagged) → DONE`
- **IN PROGRESS:** PO has dispatched build work on a short-lived slice branch.
- **VERIFY:** build complete; the independent `VERIFY-<slice>.md` + class-appropriate `machine_probe` runs on neutral CI; binding floor must be green. The PO checks the result against the DoD. Worker self-assessment is **not** completion.
- **FOUNDER REVIEW:** only for sprints touching a Class-C/`data_class`-keyed boundary (see each sprint's *founder checkpoint*); the founder authorizes the irreversible/sensitive act.
- **DONE:** PO confirms DoD met + required evidence captured. Legacy retirement (where applicable) follows the flag-OFF→30-day→archive discipline.
**No work reaches production except through the DRB release path:** ephemeral local/preview build → binding CI (neutral hardware) → non-author CODEOWNERS merge → (P2.1+) D7-gated staging-first promotion → prod canary + binding health gate. Production must remain operational at every step.
