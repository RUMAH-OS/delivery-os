# Portfolio Doctrine Seed (noun-free — copied to `memory/doctrine/` at scaffold)

> **This file is the day-1 inheritance mechanism**: every lesson below was purchased by a recorded incident
> or success in a real project, is statable with **no project noun**, and travels to every new company at
> scaffold time so it is never re-purchased by incident. Each entry cites its case study
> (`delivery-os/case-studies/`). Add to your copy via the write-back step; promote new entries to the base
> seed only through Governance §14. *(Tier 1 of three-tier memory: doctrine here · project nouns in
> `memory/<project>/` · state never stored, always derived.)*

| # | Doctrine (one line, mechanism-phrased) | Earned by (case study) |
|---|---|---|
| D-EVIDENCE | **Evidence-before-constraint**: never let a single weak probe become a stated product constraint — verify actual runtime state before calling anything missing/broken. | `case-studies/2026-06-incident-ledger.md#incident-5` |
| D-NORTHSTAR | **Conversations-not-data** (north-star class): success is measured at the end of the chain (replies → meetings → customers), never at data/supply volume; at every funnel review, follow the chain all the way. | `case-studies/2026-06-incident-ledger.md#metrics` |
| D-OVERASK | **Decide-don't-overask + pre-framed decision packages**: specialists choose safe defaults; only genuine tradeoffs escalate, batched, each pre-framed with evidence + a recommended default + a checkbox verdict — the single most effective founder-workload reduction, witnessed independently twice. | `case-studies/2026-06-incident-ledger.md#decision-packages` |
| D-PRESERVE | **Preservation-before-extraction**: a conversation/record's value is the intelligence it produces, independent of conversion — capture raw and forever; extraction layers come later and re-run. | `case-studies/2026-06-incident-ledger.md#emergent-assets` |
| D-DESTRUCT | **Self-inflicted data destruction is the most expensive defect class**: under an append-only doctrine, audit every UPDATE-clobber and CASCADE path explicitly — those losses are permanent and invisible to green tests. | `case-studies/2026-06-incident-ledger.md#n13-self-destruction` |
| D-GATES | **Gates change by signature only**; every gate gets an owner, a live counter, and a reconsideration date — a gate without these fails silently by never firing or gets quietly reinterpreted. | `case-studies/2026-06-incident-ledger.md#incident-10` |
| D-AV | **Author ≠ verifier is identity-level law**: the one rejection that mattered came from a verifier-constructed probe after the author's 57 tests passed green. The mechanism (hooks/CODEOWNERS) is base; the invariant is doctrine. | `case-studies/2026-06-incident-ledger.md#s37` |
| D-MECH | **Mechanism-over-prose**: trust in an OS link ∝ its mechanical fraction — every recorded incident occurred at a prose link; zero at a program link. A rule without a scan/fixture/hook is a wish. | `case-studies/2026-06-incident-ledger.md#the-split` |
| D-SPINE | **Deterministic-spine-first**: scores, money math, and state machines are computed in code — deterministic, reproducible, never by an LLM; the probabilistic layer rides on the unchanged spine. Held 100% in both implementations. | `case-studies/2026-06-incident-ledger.md#held-invariants` |
| D-OWNER | **Ownership-as-mechanism**: every ownership rule shipped with an enforcement (scan, client pin, append-only) has a zero-violation record; every prose-only rule drifted within ~48h. "Reads are truth, events are triggers." Apply the single-writer test to every entity. | `case-studies/2026-06-incident-ledger.md#ownership` |
| D-CONTRACT | **Contracts + probes, never narrative claims**: cross-system state travels as versioned contracts and verifiable probes — every relayed-prose claim in the record contained a false statement; every contract+token request worked first time. | `case-studies/2026-06-incident-ledger.md#relayed-prose` |
| D-AUDIT | **Audit-before-assume**: any plan gating on a peer system carries a same-day read-only audit citation of that system's actual disk — never a registry/router claim (days were once planned behind a gate that was already open). | `case-studies/2026-06-incident-ledger.md#incident-5` |
| D-REVERSIBLE | **Cheapest-reversible-version + pre-registered reversal**: behavioral surfaces ship as the cheapest reversible probe with the reversal named in advance — three pivots in the record each cost one mount change because the reversal was pre-registered. Being wrong must be cheap. | `case-studies/2026-06-incident-ledger.md#reversals` |
| D-RECOVER | **Prefer the recoverable direction**: under-merge is recoverable, over-merge co-mingles history; keep-separate over merge, preserve over normalize, import-with-marker over rewrite. | `case-studies/2026-06-incident-ledger.md#migration-fidelity` |
| D-REHEARSE | **Rehearsal-then-production with identical tooling**: same loader, same data, same battery, same day — "the production run must be boring." Rehearsal acks are labeled rehearsal-only and never satisfy production. | `case-studies/2026-06-incident-ledger.md#cutover` |
| D-DERIVE | **Cross-derive one number by hand**: in any reconciliation, at least one figure is independently hand-derived from raw source data — "green checks you didn't cross-derive are belief, not knowledge." | `case-studies/2026-06-incident-ledger.md#cutover` |
| D-REFUSE | **Refusal gates around irreversibles**: encode ratified decisions as machine-enforced refusals (an explicit `--ack` flag) — the operator-side twin of the verify-gate; it converted "did we agree?" from memory into mechanism and caught a real future-money bug. | `case-studies/2026-06-incident-ledger.md#decision-packages` |
| D-SURFACE | **Decision-surfacing is the product**: ambiguity becomes a pre-framed founder decision, never a silent default — and never a padded list; every fake decision costs trust. | `case-studies/2026-06-incident-ledger.md#decision-packages` |
| D-LONGPOLE | **Founder acts are the long pole**: credentials, DNS, accounts, and approvals are named in the slice doc at spec time and requested day one — they bottleneck more slices than engineering does. | `case-studies/2026-06-incident-ledger.md#long-pole` |
| D-CUTOVER | **Operating begins at a named cutover moment**: declare it; the declaration fires the readiness machinery. One implementation named it and the machinery ran; the other drifted into operating and its readiness skill never fired. | `case-studies/2026-06-incident-ledger.md#cutover` |
| D-ENV | **Verify the environment before DDL, every time** — schema was once applied into a live legacy production project on an unverified assumption about "the fresh project." | `case-studies/2026-06-incident-ledger.md#migration-fidelity` |
| D-PLATFORM | **Platform-conventions-first**: a 15-minute pass over the platform's documented auto-detection/connection/auth behavior before the first integration attempt — three deploy cycles were lost to documented platform behavior. | `case-studies/2026-06-incident-ledger.md#platform` |
| D-ONEDERIVE | **One derivation, many consumers**: consumers get versioned projections of the SAME functions the owner's surface uses; a consumer needing new logic means a new owner-side projection, never consumer-side reimplementation (a cross-system seam once needed zero new business logic). | `case-studies/2026-06-incident-ledger.md#ownership` |
| D-EVENTS | **Events for deltas + reads for truth**: events trigger, reads decide — the hybrid that both retrospectives independently call the one unambiguously correct cross-system decision of the period (credit: the founder's own observation, recorded as such). | `case-studies/2026-06-incident-ledger.md#n15-founder-hybrid` |
| D-NARRATE | **LLM-narrates-tools**: operational facts (amounts, dates, statuses, counts, existence) are ALWAYS tool calls; figures render from tool JSON with model prose around them — a hallucinated number then has no path to the screen. | `case-studies/2026-06-incident-ledger.md#relayed-prose` |
| D-CREDS | **Credentials are a stop condition**: a missing credential stops the lane; the platform permission classifier is honored as a fourth enforcement layer (a refused token mint and two refused premature production actions were each correct). | `case-studies/2026-06-incident-ledger.md#long-pole` |
| D-UNPROVEN | **A packaging/process conversion is unproven until it fires on real work** (N1): practices earn trust by firing; projected savings are not measurements. Name the adoption moment that will test it. | `case-studies/2026-06-incident-ledger.md#n1-unproven-packaging` |
